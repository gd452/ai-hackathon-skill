# 라이브 데모 스크립트 (7분)

> 노트북 화면을 띄우고 한 번에 흐름을 보여주는 용도. 청중 / 회의 / 발표 어디서나 재사용.

## 0. 셋업 (시작 전 30초)

- Claude Code 또는 Cursor 를 띄워둠
- `~/Development/ai-hackathon-skill/` 열어둠
- README + SKILL.md 띄워두면 좋음

## 1. 도입 (30초)

> "AI 시대에 해커톤을 1인이 운영할 수 있게 만든 스킬 번들입니다. AI 가 만든 걸 AI 가 채점하는 풀 사이클을 한 번에 보여드릴게요."

`SKILL.md` 첫 화면 (description 부분) 보여주기.

## 2. 입력 — 한 줄로 끝 (30초)

`examples/sample_input.md` 첫 5줄 복사:

```
주제: AI 시대의 1인 사이드프로젝트 — 8시간 안에 동작하는 무엇이든
기간: 8시간
참가자 수: 30명 (개인전)
도구 제약: 자유
심사 기준 가중치: README 20%, 코드 25%, AI 활용도 25%, 완성도 20%, 차별성 10%
```

> "이 5줄 입력으로 5개 산출물이 한 번에 나옵니다."

## 3. 산출물 #1 — 해커톤 SPEC (1분)

`examples/sample_output_spec.md` 열어서:
- 일정 표 (등록·개회·해킹·데모·심사·시상)
- 심사 기준 + 가중치
- 운영 체크리스트 (사람이 채울 빈칸)

> "기획서가 이렇게 한 번에 나오고, 사람이 채워야 하는 항목 (장소·식사·전원) 은 명시적으로 빈칸으로 둡니다. AI 가 모르는 영역은 정직하게 비워두는 게 핵심."

## 4. 산출물 #2 — 온보딩 메시지 (1분)

`examples/sample_output_onboarding.md`:
- Slack / Telegram / 이메일 3가지 변형

> "참가자 안내를 채널별로 자동 생성. 톤이 약간씩 다르죠 — Slack 은 이모지, Telegram 은 ASCII, 이메일은 정중. 30명한테 일일이 메시지 쓰는 시간이 사라집니다."

## 5. 산출물 #3 — AI 채점 (2분, 핵심)

`examples/sample_output_rubric.md`:
- Team #03 채점 결과 펼쳐서

> "이게 AI 채점기예요. README · 코드 · AI 활용도 · 완성도 · 차별성 5개 축으로 0~10 점수 매기고, 각 항목마다 한 줄 코멘트. 중요한 건 **사람 심사위원의 의사결정을 대체하는 게 아니라 1차 정렬 보조** 라고 매번 명시한다는 점."

> "특히 차별성 항목은 'AI 는 가설만 제시, 사람 평가 권장' 이라고 못 박아둠. 책임 경계를 명확히."

## 6. 산출물 #4 — 리더보드 (1분)

`examples/sample_output_leaderboard.md`:
- 종합 순위 표
- 부문별 1위
- 사람 심사 권고 (상위 N팀)

> "리더보드는 종합 점수 외에 부문별 1위도 자동 분리. 그리고 'AI 가 의심스러워하는 항목' 을 사람 심사 권고 섹션에 모아둡니다 — 영상 4분 초과 같은 운영 규정 위반은 사람이 결정해야 하니까."

## 7. 마무리 — 어떻게 만들었나 (1분)

`BUILD_LOG.md` 보여주기:
- 짧은 시간 안에 SKILL.md + 5 템플릿 + 4 예시 출력 완성
- Claude Code (이 자리에서) + OpenAI (외부 시각 검증) 병렬 사용
- 외부 호출 = LLM API 한정 — Anthropic Claude Skills 표준

> "처음부터 끝까지 한 자리에서. 이게 AI 시대 해커톤 운영의 한 단면이에요. 이 스킬 자체가 다음 해커톤에서는 입력이 됩니다 — 누군가 이 스킬을 fork 해서 자기 행사에 맞게 가중치만 바꾸면 끝."

## 예상 Q&A

**Q: 실제 코드 채점은 어떻게 돌아가요?**
A: 스킬은 prompt + template + LLM API 호출만. 실제 GitHub repo fetch 와 정적 분석은 호출 환경 (Claude Code, Cursor, Continue) 의 다른 도구가 담당. 스킬은 어떤 신호를 보고 어떤 식으로 점수 매길지 정의.

**Q: 사람 심사위원 없이 운영 가능한가?**
A: 아니요. 차별성·운영 규정 위반·시상 결정은 사람 영역. 80% 자동화가 목표 — 100% 가 아님.

**Q: 한국어 외에 다른 언어 지원?**
A: 템플릿이 다 markdown 이라 prompt 한 줄 추가로 다국어 가능. 차별점은 한국어 운영 환경 / 한국 IT 컨텍스트.

**Q: 보안은?**
A: 외부 호출 = 3 모델 LLM API 한정. 사용자 본인의 환경에서만 동작. 참가자 본명·연락처는 채점 단계에서 익명화 권장 (skill 안에 명시).
